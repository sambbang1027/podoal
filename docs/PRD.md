# 🍇 포도알 파이터 (Podobal Fighter) — PRD

> **한 줄 정의:** 멜론티켓의 실제 예매 프로세스와 패널티 메커니즘을 완벽히 시뮬레이션하여
> 유저의 피지컬과 타이밍을 훈련시키는 실전형 티켓팅 트레이너 Web App.

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 개발 기간 | 2일 (초단기 MVP) |
| 타겟 플랫폼 | 데스크톱 웹 브라우저 (PC 전용, 모바일 최적화 없음) |
| 기술 스택 | Vite + React + Tailwind CSS + Zustand |
| 백엔드 | 없음 — 전적으로 클라이언트 사이드 + LocalStorage |
| 디자인 방향 | Tailwind 유틸리티만으로 사이버틱한 픽셀 UI. 외부 그래픽 에셋 없음 |

## 2. 전역 아키텍처

### 2-1. 난이도 시스템 (maxQueueSize)

세팅 단계에서 입력된 `maxQueueSize` 값이 **모든 스테이지의 난이도 기준값**이 된다.

| 프리셋 | 명칭 | maxQueueSize |
|--------|------|-------------|
| 소규모 | 소규모 | 2,000 |
| 중대규모 | 중대규모 | 30,000 |
| 지옥문 | 지옥문 | 100,000 |
| 직접 입력 | — | 1,000 ~ 150,000 |

### 2-2. 전역 상태 (Zustand Store)

```ts
{
  // 세팅
  maxQueueSize: number,
  difficulty: 'small' | 'medium' | 'hell' | 'custom',

  // 스테이지별 기록
  reactionTime: number,      // ms — 정각 진입 오차
  popupClickTime: number,    // ms — 팝업 3콤보 소요 시간
  captchaTime: number,       // ms — 보안문자 입력 소요 시간
  totalTime: number,         // ms — 예매하기 클릭 ~ 보안문자 완료

  // 파생 값
  queueNumber: number,       // 부여된 대기 번호
  alreadyTakenCount: number, // 이선좌 당한 횟수

  // 결과
  rank: 'S' | 'A' | 'B' | 'F',

  // 액션
  setDifficulty, setReactionTime, setPopupClickTime,
  setCaptchaTime, setTotalTime, setRank, reset,
}
```

### 2-3. 라우팅 구조

```
/           → SETTING  (Step 0)
/ready      → READY    (Step 1)
/popup      → POPUP    (Step 2)
/queue      → QUEUE    (Step 3)
/seat       → SEAT     (Step 4)
/captcha    → CAPTCHA  (Step 5)
/result     → RESULT   (Step 6)
```

### 2-4. F5 패널티 — 글로벌 이벤트 바인딩

앱 최상위에서 `keydown` 이벤트 리스닝.
F5 또는 Ctrl+R 감지 시:

1. `preventDefault()` — 실제 새로고침 차단
2. z-index: 9999 흰 화면 오버레이 렌더링
3. "서버 재연결 중..." 스피너 1.8초 표시
4. 오버레이 해제 — 사용자는 타이밍을 놓친 상태

> 적용 구간: READY, POPUP, QUEUE 스테이지에서만 활성화.
> SETTING, RESULT에서는 F5 패널티 없음.

## 3. 스테이지별 상세 스펙

---

### Step 0. SETTING — 난이도 설정

**목적:** 훈련 경쟁률 설정. 이 값이 이후 모든 스테이지를 지배한다.

**UI 구성**
- 가상 콘서트 이름 입력 (선택, placeholder: "예: aespa WORLD TOUR")
- 대기 인원수 직접 입력 (1,000 ~ 150,000, 숫자 입력 필드)
- 프리셋 버튼 3개: `소규모 (2,000명)` / `중대규모 (30,000명)` / `지옥문 (100,000명)`
- [훈련 시작] 버튼 → `/ready` 이동

**예외 처리**
- 범위 벗어난 입력 시 인라인 에러 메시지 표시, 이동 불가

---

### Step 1. READY — 정각 타임어택

**목적:** 서버 시계 00.000에 [예매하기] 버튼을 정확히 누르는 반응속도 훈련.

**UI 구성**
- 상단: 카운트다운 시작 시점 셀렉터
  - `[10초 전 시작]` / `[20초 전 시작]` / `[30초 전 시작]`
  - 선택 즉시 카운트다운 시작
- 중앙: `HH:MM:SS.mmm` 형식의 밀리초 단위 서버 시계 (0.001초 단위 갱신)
- 하단: [예매 준비중] 버튼 — 정각 전 완전 비활성화 (커서 올려두기 유도)

**정각 처리**
- 카운트다운 00.000 도달 시 버튼이 초록색 [예매하기]로 전환
- 유저 클릭 순간 `reactionTime = 클릭 timestamp − 정각 timestamp` (ms) 기록
- `/popup` 이동

**F5 패널티 (이 구간 활성)**
- 정각 전 F5/Ctrl+R → 흰 화면 오버레이 1.8초 → 타이밍 강제 손실

---

### Step 2. POPUP — 날짜/회차 스피드런

**목적:** 날짜 → 회차 → 다음단계의 3콤보 연속 클릭 피지컬 훈련.

**UI 구성**
멜론티켓 팝업 스타일 모달:

```
┌─────────────────────────────┐
│  날짜 선택                   │
│  [5/17 토] [5/18 일] ← 활성  │
│                             │
│  회차 선택 (날짜 클릭 후 표시) │
│  [17:00] [20:00] ← 클릭 대상 │
│                             │
│         [다음단계 →]         │
└─────────────────────────────┘
```

**인터랙션 플로우**
1. 날짜 1개 클릭 (비활성 날짜는 흐릿하게)
2. 날짜 클릭 후 회차 선택지 노출 → 회차 클릭
3. [다음단계] 버튼 활성화 → 클릭 시 `/queue` 이동

**기록**
- `popupClickTime`: 팝업 모달 등장 시점 ~ [다음단계] 클릭 시점까지의 총 ms

**대기 순번 계산**
```
queueNumber = Math.floor(
  maxQueueSize * (reactionTime / 1000) * (popupClickTime / 3000)
)
// 최솟값: 1, 최댓값: maxQueueSize
```

---

### Step 3. QUEUE — 가상 대기열

**목적:** 실시간으로 줄어드는 대기 번호를 보며 인내하는 심리 훈련.

**UI 구성**
- 중앙 대형 숫자: `내 앞 대기자 수: 12,847명`
- 진행 바: 전체 대비 현재 남은 비율
- 문구: "잠시만 기다려주세요. 앞의 고객님들의 예매가 진행 중입니다."

**대기열 감소 로직**
```
interval: 500ms ~ 1500ms (불규칙 난수)
1회 감소량: 1 ~ Math.floor(maxQueueSize * 0.002) (난수)
→ 불규칙하게 줄어드는 느낌 연출
```
큐가 0이 되면 자동으로 `/seat` 이동

**F5 패널티 (이 구간 활성)**
```
F5/Ctrl+R → queueNumber += Math.floor(maxQueueSize * 0.3)
→ "⚠️ 새로고침으로 인해 대기자 N명이 추가되었습니다." 팝업 표시
```

---

### Step 4. SEAT — 포도알 서바이벌

**목적:** 실시간으로 사라지는 좌석에서 빈 자리를 클릭해 결제 단계로 넘어가는 핵심 스테이지.

**UI 구성**
- 20×20 그리드, 총 400석
- 상태별 색상:
  - 🟣 보라색: 선택 가능한 빈 좌석 (클릭 가능)
  - ⬜ 회색: 이미 선점된 좌석 (클릭 불가)
  - 🟡 노란색: 유저가 선택 중인 좌석

**봇 시뮬레이션 (난이도별)**

| 난이도 | 봇 수 | 삭제 간격 |
|--------|-------|----------|
| 소규모 | 3개 | 400ms ~ 800ms |
| 중대규모 | 8개 | 200ms ~ 500ms |
| 지옥문 | 20개 | 50ms ~ 200ms |

각 봇은 독립적인 `setInterval`로 랜덤 빈 좌석을 회색으로 전환.

**이선좌 확률 폭탄 (난이도별)**

| 난이도 | 이선좌 확률 |
|--------|------------|
| 소규모 | 15% |
| 중대규모 | 30% |
| 지옥문 | 45% |

유저가 보라색 좌석 클릭 시 위 확률로 모달 발동:
```
┌──────────────────────────────┐
│  이미 선택된 좌석입니다.       │
│  다른 좌석을 선택해 주세요.    │
│              [확인]           │
└──────────────────────────────┘
```
Enter 또는 [확인] 클릭으로 닫음. `alreadyTakenCount += 1`

이선좌 없이 클릭 성공 시 → `/captcha` 이동

**실패 조건**
400석 전부 회색이 되면 "매진되었습니다." 화면 → `/result` 이동 (실패 처리)

---

### Step 5. CAPTCHA — 보안문자 타이핑

**목적:** 6자리 대문자 보안문자를 오타 없이 빠르게 입력하는 최종 관문.

**UI 구성**
- 기울어진 스타일의 영문 대문자 6자리 이미지 (CSS로 구현)
- 입력 필드 `autoFocus`, 6자리 대문자만 허용
- 잔여 시간 카운트다운: **5초**
- 타이머 바 (점점 빨간색으로)

**로직**
- 입력 즉시 대소문자 무시하고 6자리 일치 여부 검증
- 5초 초과 → 실패 → `/result` 이동 (시간 초과 처리)
- 오타 제출 → "보안문자가 일치하지 않습니다." 인라인 에러, 새 문자 생성, 타이머 리셋
- 성공 → `captchaTime` 기록 → `/result` 이동

---

### Step 6. RESULT — 전투 분석 대시보드

**목적:** 세션 데이터 분석, 등급 부여, 역대 기록 비교, 개인화 피드백 제공.

#### 6-1. 등급 산정 로직

```ts
function calcRank(reactionTime, popupClickTime, captchaTime): 'S' | 'A' | 'B' | 'F' {
  // S: 신의 손
  if (reactionTime < 50 && popupClickTime < 1500 && captchaTime < 2000) return 'S'
  // A: 광클 장인
  if (reactionTime < 200 && popupClickTime < 3000 && captchaTime < 3500) return 'A'
  // B: 평범한 손가락
  if (reactionTime < 500) return 'B'
  // F: 취소표나 노려라
  return 'F'
}
```

#### 6-2. 등급 라벨

| 등급 | 칭호 |
|------|------|
| S | 🏆 신의 손 |
| A | ⚡ 광클 장인 |
| B | 🖐️ 평범한 손가락 |
| F | 🙏 취소표나 노려라 |

#### 6-3. 역대 기록 탑 3

```
🥇 1위: 2.15초 (랭크 S) - 2026-05-12
🥈 2위: 2.80초 (랭크 A) - 2026-05-14
🥉 3위: 3.10초 (랭크 A) - 방금 한 게임! (NEW!)
```

#### 6-4. 개인화 피드백 메시지

이전 세션과 각 지표를 비교해 스테이지별 1줄 코멘트 생성.

```
예시:
"보안문자 입력이 이전 판보다 0.4초 단축됐어요! 
 하지만 정각 클릭이 230ms 늦었습니다. 시계에 집중하세요."
```

비교할 이전 세션 없을 경우: "첫 훈련 기록이에요! 다음 판에서 점수를 비교해 드릴게요."
